What a merge conflict looks like
<<<<<<< HEAD (your current changes) public int getTotal() { return price * quantity; } ======= public int getTotal() { return price * quantity * discount; } >>>>>>> teammate-branch (incoming changes)
How to resolve it, step by step
Best prevention: pull before you start working, keep commits small and focused, and communicate with teammates about which files you're editing.
From Professor Chris's guidelines
✓ Do
Pull before starting. Always run git pull first so you're working on the latest version.
Commit regularly. Small, frequent commits are easier to understand and undo.
Write descriptive messages. Add validation to Booking constructor beats update code.
Communicate with teammates. Say which file or class you're editing before you start.
Integrate early. Merge and test your teammate's code well before the deadline.
Push regularly. Run git push after commits so teammates have your latest work.
✗ Don't
Don't push broken code. Make sure it compiles and runs before pushing.
Don't bundle unrelated changes. Keep commits focused — one logical change per commit.
Don't wait until the last day. Integration issues appear when code combines — find them early.
Don't edit the same file simultaneously. Coordinate with your teammate to avoid unnecessary conflicts.
